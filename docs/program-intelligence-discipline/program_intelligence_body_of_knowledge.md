# Program Intelligence Body of Knowledge (PIBOK)

This document defines the **Program Intelligence Body of Knowledge
(PIBOK)**.\
PIBOK structures the Program Intelligence discipline into its primary
knowledge domains and describes how these domains interact to support
evidence‑based understanding of program execution.

The PIBOK serves as the structural backbone of the **Program
Intelligence Handbook** and defines the core knowledge areas of the
discipline.

------------------------------------------------------------------------

# 1. Purpose

The purpose of the Program Intelligence Body of Knowledge is to organize
the discipline of Program Intelligence into a coherent and structured
set of knowledge domains.

Each domain represents a fundamental capability required to observe,
interpret, and understand program execution.

Together these domains form the conceptual architecture of the
discipline.

------------------------------------------------------------------------

# 2. Conceptual Background

Modern delivery programs operate as complex socio‑technical systems
composed of:

• people and teams\
• delivery processes\
• technology systems\
• operational environments

Execution activities across these systems generate large volumes of
data.

However, raw execution data does not automatically translate into
understanding.

Program Intelligence provides the frameworks required to transform
execution data into:

• observable signals\
• interpretable delivery patterns\
• evidence‑based insights for leadership

The PIBOK organizes these capabilities into distinct knowledge domains.

------------------------------------------------------------------------

# 3. PIBOK Domain Structure

The Program Intelligence Body of Knowledge consists of the following
domains:

1.  Program Intelligence Foundations\
2.  Program Execution Observability\
3.  Execution Signal Framework\
4.  Signal Interpretation and Pattern Analysis\
5.  Program Intelligence Analytics\
6.  Executive Intelligence and Decision Support\
7.  Program Intelligence Governance\
8.  Relationship of the Discipline to Other Streams

Each domain contributes to the overall capability of observing and
interpreting program execution.

# 12. Interpretation and Implications

The PIBOK transforms Program Intelligence from an idea into a structured
discipline.

By defining its knowledge domains, the discipline gains:

• conceptual clarity\
• structured learning paths\
• standard vocabulary\
• shared analytical models

Over time, the PIBOK may evolve to include additional domains as the
discipline matures.
