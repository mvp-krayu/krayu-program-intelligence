The situation
INVESTIGATE decision state triggered

The LENS Assessment review concluded with an INVESTIGATE state. A structural evidence gap is identified — but the assessment artifact does not surface which dimensions are blocked, where evidence is absent, or what would resolve the gap.

Diagnostic Access surfaces that layer. Each unknown dimension is named. Each scope gap is described. Each system area is identified.

What you receive
  • Diagnostic zone inventory — deterministic, derived from canonical topology and signal registry
  • Per-dimension: named, classified, severity and confidence assigned
  • Scope gap per dimension — evidence type absent that blocks confirmation
  • System area per dimension — functional area responsible
  • What evidence would move each dimension from unknown to confirmed
  • Diagnostic Narrative — HTML export with structural evidence topology graph

What this gives you

A named, evidence-bounded view of each structural unknown in the assessed environment. Dimensions are identified by functional label. Scope gaps are described by evidence type absent. System areas are identified by functional name. This is not analysis — it is the evidence gap layer surfaced and bounded to what the canonical data directly supports.

When to use

When the LENS Assessment produced an INVESTIGATE decision state and you need to understand the evidence gap structure before deciding the next engagement step. Diagnostic Access names and bounds the investigation. It does not resolve it — that requires targeted evidence collection or escalation.

Interaction modes
WHY Phase 1 — Active
Zone classification rationale — which canonical facts triggered zone type assignment. Returns: grounding state, signal count, confidence, structural scope.
EVIDENCE Phase 1 — Active
Available trace links, missing evidence inventory, signal coverage summary per zone.
TRACE Phase 2 — Deferred
Full propagation traversal from zone anchor domain. Depth ≤ 2 adjacency. Not available in Phase 1.
Boundary constraint
inference_prohibition: ACTIVE

All outputs are bounded to what the evidence set directly supports. The workspace produces no inferences about root causes, resolution paths, or forward projections. This applies to all outputs, all interaction modes, and all exports.

What it does not do
  • No advisory direction, prioritisation guidance, or investigation sequence
  • No root cause claims or forward projections
  • No certainty claims about current degradation state
  • Consequence class per dimension deferred to Phase 2
  • TRACE mode (full traversal) deferred to Phase 2
  • No internal system identifiers in any output
