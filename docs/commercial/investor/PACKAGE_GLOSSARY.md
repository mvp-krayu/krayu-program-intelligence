# Program Intelligence — Package Glossary

**CONFIDENTIAL — KRAYU BV — June 2026**

</div>

---

This glossary defines the key terms used across the Program Intelligence commercial package. Terms are listed in the order a new reader encounters them.

---

| Term | Definition |
|------|-----------|
| **Program Intelligence** | A governed cognition runtime that transforms structural evidence into qualified understanding. The core product and category created by Krayu. |
| **Governed Cognition** | Intelligence output that carries explicit qualification metadata — confidence level, evidence lineage, governance boundaries, and authority limits. Distinguished from unqualified AI output or consulting opinion. |
| **Evidence Lineage** | The complete, traceable chain from a specific structural artifact (file, dependency, class) through every derivation step to the final executive conclusion. Any output can be traced back to the evidence that produced it. |
| **PICP** | **Program Intelligence Cognition Package.** The portable, versioned artifact that the runtime produces after qualifying evidence. Contains governed cognition objects with full metadata. The PICP is the interface between intelligence production and intelligence consumption. |
| **Cognition Object** | A governed intelligence artifact produced by the runtime — such as a structural posture assessment, a tension map, or an exposure evaluation. Each object traces to structural evidence and carries qualification status. |
| **EIR** | **Executive Intelligence Report.** A governed, multi-chapter intelligence report synthesized from PICP cognition objects. The first commercial output format. The BlueEdge EIR (Document 2 of this package) is a proof specimen. |
| **LENS** | The interactive intelligence workbench that projects PICP cognition across multiple cognitive personas (executive, operational, structural, forensic, investigative). Used for real-time structural exploration. |
| **Domain Module** | An evidence processing module for a specific evidence category (e.g., Software Intelligence). Domain modules ingest evidence and contribute cognition objects to the PICP. Additional domain modules expand the runtime's evidence coverage. |
| **S-Level** | **Qualification Stage.** The maturity progression of a governed specimen: S0 (Intake) → S1 (Structured) → S2 (Governed) → S3 (Certified). Each level is earned through evidence, not declared. Progression is irreversible and auditable. |
| **Q-Class** | **Quality Classification.** The grounding strength of qualified cognition: Q-01 (Structural + Semantic + Grounded) through Q-04 (Ungrounded). Determines the authority ceiling — what the runtime is permitted to conclude. |
| **Structural Condition** | A qualified pattern detected in structural evidence — such as pressure convergence, dependency bottleneck, propagation asymmetry, or cross-domain coupling. Conditions have executive meaning, not just technical significance. |
| **Governance Boundary** | The explicit limits on what the runtime may conclude at a given qualification level. Includes authority ceilings, absolute prohibitions, and non-automatable decision gates. Boundaries are part of the output, not hidden constraints. |
| **Specimen** | A real system that has been ingested, analyzed, qualified, and projected through the full Program Intelligence governance lifecycle. BlueEdge is the primary proof specimen. |
| **Consumer** | Any system or output format that receives PICP cognition through the projection architecture. Examples: EIR (report), LENS (interactive workbench), marketplace modules (third-party consumers). |
