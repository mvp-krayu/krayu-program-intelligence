## Graph Intent

Each document in the PiOS architecture corpus is a node in the Obsidian knowledge graph. Every link — whether a relative Markdown link (`./file.md`) or an Obsidian wiki link — creates a structural edge between nodes. These edges are not navigational conveniences; they reflect real authority and dependency relationships within the architecture.

The graph is deliberately layered. [[pios_whitepaper]] is the surface-level entrypoint — it links outward to every other document in the reading paths. [[program_intelligence_stack]] is the structural authority — it defines the hierarchy from discipline to system to product to module, and its edges reflect constraint direction. [[pios_technical_appendix]] is the formal reference — it links back to the canonical layer model as its root authority. [[pios_investor_narrative]] is positioned at the edge of the graph nearest the discipline, linking forward into the system documents without reaching into technical detail. [[index]] anchors the governance corpus — canonical architecture, drift register, remediation chain — and is reachable from every document in this set.

The graph is connected. No document in this set is an orphan. Every node is reachable from this root within two hops.
