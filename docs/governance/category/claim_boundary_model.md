# Claim Boundary Model

Applies to: ESI, RAG

Allowed Claims:
- derived from execution telemetry
- reflects observable patterns in execution signals
- bounded by available signal inputs
- part of a broader signal system

Disallowed Claims:
- predicts outcomes
- guarantees execution success/failure
- represents full system state
- operates independently of signal infrastructure

Mandatory Framing:
- must reference signal origin (implicit or explicit)
- must not be presented as standalone metric

Constraint:
No WEB or product surface may exceed these boundaries.
