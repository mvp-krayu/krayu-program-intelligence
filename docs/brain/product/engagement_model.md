# engagement_model

## What Is Being Offered

A structured, time-bounded assessment engagement producing a single governed intelligence artifact. Four steps. Minimal client burden. One deliverable plus a guided review session.

---

## Engagement Steps

| Step | Name | System Basis | Client Effort |
|---|---|---|---|
| 1 | Intake | [[../canonical/streams/PRODUCTIZE.RAW.SOURCE.INTAKE.01]] | Low — context provision only |
| 2 | Analysis | 40.2 → 40.3 → 40.4 → PiOS Core | None — system-executed |
| 3 | Report | [[../canonical/streams/PRODUCTIZE.LENS]] | None — artifact generated |
| 4 | Review | Guided session using [[lens_report]] | Medium — review and decision |

---

## Evidence Supporting This Model

- Intake stream is governed and bounded — [[../canonical/streams/PRODUCTIZE.RAW.SOURCE.INTAKE.01]]
- Analysis chain is deterministic — [[../canonical/04_INVARIANTS]] INV-06
- Report is structurally generated, not narratively composed

---

## Constraints

- No instrumentation required from client
- No extended access required
- Engagement produces exactly one governed report artifact
- Review session is included in Tier 1 (LENS Assessment)

---

## Product Ladder Integration

Step 4 (Review) produces a decision state: proceed / investigate / escalate

- "Investigate" → Tier 2 (Diagnostic Access)
- "Escalate" → Tier 3 (Enterprise Access)

See: [[01_PRODUCT_MAP]]

---

## What Is Measurable

- Engagement duration (bounded, single sprint)
- Deliverable count: 1 report + 1 review session
- Decision output: one of three defined states
