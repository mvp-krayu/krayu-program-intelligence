### 5.3 Executive Intelligence Report Model

**HEADER ZONE:**
- Readiness Badge: maps to Q-00 (FULL), Q-01 (PRIMARY QUALIFIED), Q-02 (PARTIAL), Q-03 (DIAGNOSTIC), Q-04 (SUPPRESSED)
- Executive Readiness label: uses ALI aliased names (not technical keys)
- Topology scope: domain count, cluster count (as client-appropriate numbers)
- Report date and baseline reference

**NARRATIVE ZONE:**
- All narrative output passes through cognitive normalization pipeline
- Technical cluster/domain labels → human-readable aliased names (ALI-01..07)
- Qualifier labels applied consistently (Q-00..Q-04)
- Executive tone: structural, evidence-first, non-speculative
- WHY section: primary causal chain in plain executive language
- Structural summary: key topology findings ranked by pressure

**EVIDENCE ZONE:**
- Evidence panels per domain: what was found, what it means
- Signal cards: one per active signal; CPI/CFA rendered as normalized labels
- Qualifier indicators: inline state labels (PARTIAL GROUNDING, FULL EVIDENCE, etc.)
- Propagation markers: shows where pressure originates and flows

**TRACE ZONE (collapsed by default, expandable on request):**
- Propagation path: cluster-to-cluster pressure flow (simplified topology)
- Signal derivation: which clusters contributed, normalized values shown as relative indicators
- Lineage reference: stream and baseline reference for audit purposes
- No raw TAXONOMY-01 field values in executive surface

### 5.4 Semantic Rendering Strategy

Semantic rendering applies cognitive normalization rules at the rendering layer (never at the derivation layer):

| Raw field | Rendering rule | ALI reference |
|-----------|----------------|---------------|
| `cpi_score` | Rendered as "Cluster Pressure Level" | ALI-01 |
| `cfa_score` | Rendered as "Fan Distribution Asymmetry" | ALI-02 |
| `executive_readiness` | Rendered per Q-00..Q-04 label | ALI-03 |
| Domain technical ID | Client-configured domain alias | ALI-04 |
| Cluster technical key | Human-readable cluster name | ALI-05 |
| `DIAGNOSTIC_ONLY` | "Under Structural Review" | ALI-06 |
| `SUPPRESSED_FROM_EXECUTIVE` | Not rendered in executive surface | ALI-07 |

Semantic rendering constraints (hard):
- TAXONOMY-01 fields are NEVER rendered in executive surface
- `signal_value`, `activation_state`, `derivation_hash`, `signal_stable_key` are internal only
- Aliasing rules (ALI-01..07) apply at the report rendering layer only
- No aliasing occurs upstream of the rendering layer

### 6.3 Evidence Panel Architecture

**WHY Panel:**
- Purpose: answers "why does this readiness state exist?"
- Source: primary evidence envelope from evidence injection pipeline
- Content: causal chain from structural signals to readiness classification
- Rendering rule: normalized labels only; no raw TAXONOMY-01 fields
- Governance: evidence_object_hash verifies panel integrity

**EVIDENCE Panel:**
- Purpose: shows what structural facts underpin the intelligence
- Source: domain evidence records from evidence injection pipeline
- Content: per-domain signal evidence, normalized
- Rendering rule: ALI rules applied; qualifier labels from Q-taxonomy
- Governance: sealed at report generation; not live inference

**TRACE Panel:**
- Purpose: allows audit-grade derivation trace for governance review
- Source: derivation_trace from TAXONOMY-01 lineage record
- Content: simplified propagation path (cluster aliases); stream reference
- Rendering rule: no raw TAXONOMY-01 values; trace is human-readable
- Audience: internal/advisory — may be hidden from basic executive view
- Governance: lineage record authenticated by derivation_hash

### 6.4 Propagation Explainer Model

The propagation explainer renders the pressure propagation path in executive-accessible language:

```
PROPAGATION EXPLAINER:
"Execution pressure originates in [Domain A — aliased name].
Cluster [X — aliased] drives high concentration pressure (above threshold).
This propagates through [Domain B — aliased] into downstream delivery clusters.
The asymmetric fan pattern in [Domain A] indicates structural bottleneck
rather than capacity constraint."
```

Rules:
- Only aliased names appear
- No threshold values, score values, or TAXONOMY-01 fields
- Propagation direction is structural, not causal inference
- Qualifier: if PARTIAL grounding, propagation explainer carries Q-01/Q-02 label

### 6.5 Qualifier Rendering Strategy

| Qualifier | Surface Label | Rendering Treatment |
|-----------|--------------|---------------------|
| Q-00 FULLY_GROUNDED | "Full Evidence" | No qualifier chip — full confidence |
| Q-01 PARTIALLY_GROUNDED | "Partial Grounding" | Amber qualifier chip; tooltip explains |
| Q-02 STRUCTURALLY_GROUNDED | "Structural View" | Blue qualifier chip; evidence scope noted |
| Q-03 DIAGNOSTIC_QUALIFIED | "Under Review" | Grey indicator; narrative scoped accordingly |
| Q-04 SUPPRESSED | Not rendered in executive surface | Report section marked unavailable |

### 6.6 Grounding-Aware Rendering

When domain grounding is partial (Path A.5 — current state):
- Grounded domains render full evidence panels
- Ungrounded domains render STRUCTURAL_LABELS_ONLY scope note
- Report-level qualifier reflects lowest-confidence domain
- Propagation explainer scopes its claims to grounded topology only

This ensures the NextGen report is honest about grounding depth without hiding useful structural intelligence.

## 8. Intelligence UX Doctrine

### 8.1 Positioning

LENS is positioned as **executive intelligence infrastructure** — not a consumer AI tool, not a generic SaaS dashboard, not a chatbot platform.

The visual and interaction language must communicate:
- **Authority:** this is a system to be trusted with executive decisions
- **Evidence:** every claim has a source; every assertion has grounding
- **Clarity:** complex structural intelligence, made readable at executive level
- **Control:** the executive is in command; the system informs, not directs

### 8.2 Visual Design Principles

**Principle 1 — Intelligence Density Without Cognitive Overload**
Information must be dense but structured. An executive intelligence surface carries more information per screen than a consumer product — but it organizes that information in hierarchical layers: summary → evidence → trace. Never dump all layers simultaneously.

**Principle 2 — Evidence-First Visual Hierarchy**
The primary visual element is the intelligence narrative. Secondary is the evidence structure. Tertiary is the derivation trace. Visualizations support narrative — they do not replace it.

**Principle 3 — Structural Truth Over Aesthetic Appeal**
Visual design serves structural truth. Topology visualizations show what the topology actually is — not an aesthetically pleasing abstraction. Signal cards show normalized evidence — not optimistic summaries.

**Principle 4 — Readiness State Governs Color**
The readiness state classification system (Q-00..Q-04) is the primary color authority. EXECUTIVE_READY surfaces use neutral confidence tones. DIAGNOSTIC_ONLY surfaces use amber. SUPPRESSED surfaces are visually absent. Color is governance-driven, not aesthetic.

**Principle 5 — Professional Over Consumer**
No rounded corners suggesting friendliness, no animations suggesting delight. The design language is precision engineering: tight grids, explicit labels, authoritative typography, structured information hierarchy.

**Principle 6 — Operator-Grade Layout**
Layout inspiration: intelligence systems, operational command surfaces, decision-support consoles, financial terminal interfaces. Not: marketing dashboards, consumer analytics, general-purpose BI tools.

### 8.3 Interaction Philosophy

**What LENS interaction IS:**
- Evidence exploration: drill into the evidence behind a finding
- Topology navigation: move through the structural topology
- Investigation: follow a structured question chain toward understanding
- Explainability: understand why the system concluded what it concluded

**What LENS interaction is NOT:**
- Open-ended conversation with AI
- Query-response chat interface
- Natural language command surface
- Autonomous AI agent exposure

The interaction model is **interrogation-first, not conversation-first.** The executive interrogates the structure; the system responds with governed evidence. The executive does not prompt an AI; the executive navigates a structural intelligence layer.

### 8.4 Typography Strategy

- Primary typeface: monospaced or semi-monospaced for data/signal surfaces; geometric sans-serif for narrative
- Code/trace surfaces: strictly monospaced
- Signal labels: all-caps or small-caps; structured label style
- Narrative: proportional sans-serif; executive readability standards
- Qualifier labels: clear badge/chip style; color-coded per Q-taxonomy

### 8.5 Component Language

| Component | Design Language |
|-----------|----------------|
| Readiness Badge | Precise, authority-conveying badge; Q-xx taxonomy drives color |
| Signal Card | Data-dense card; normalized labels; no decorative elements |
| Evidence Drawer | Structured reveal; progressive disclosure; evidence-anchored |
| Propagation Path | Directional graph; structural, not decorative |
| Qualifier Chip | Clear label; tooltip; honest about scope limitation |
| WHY Panel | Structured narrative block; not a conversational bubble |
| Topology Map | Precision diagram; read-only; no interactive editing |
| Explainability Sidebar | Structured panels; authority-conveying; not a chat window |

### 8.6 What to Explicitly Avoid

| Anti-Pattern | Why Forbidden |
|-------------|--------------|
| Chat bubbles | Implies conversational AI positioning (wrong phase) |
| "Ask me anything" inputs | Free-form AI interaction not authorized until Phase 5 |
| Loading spinners with AI messaging ("thinking...") | Implies uncontrolled AI process |
| Confidence percentages without grounding proof | Implies probabilistic inference claiming evidence authority |
| Green = good, red = bad simplification | Oversimplifies nuanced readiness states |
| Empty panels with placeholder text | No empty panels (rule inherited from UI governance) |
| Hidden state (no silent fallback) | Invalid states must be visible (UI governance rule) |
| Decorative data visualizations | Aesthetics must not override structural truth |

### 8.7 Design System Direction

The LENS design system is built around four design tokens:

| Token | Meaning |
|-------|---------|
| `--intelligence-authority` | Deep structural truth color — primary tone for authoritative surfaces |
| `--evidence-confirmed` | Full-grounding color — used only when Q-00 (fully grounded) |
| `--evidence-qualified` | Partial evidence tone — Q-01/Q-02 qualified states |
| `--evidence-diagnostic` | Under-review tone — Q-03/Q-04 states |

Typography scale anchors:
- `--type-executive` — primary narrative; executive readability
- `--type-signal` — signal labels; structured data
- `--type-trace` — derivation trace; monospaced precision

Spacing grid: 4px base grid; 8/16/24/32px rhythm; tight information density without crowding.
