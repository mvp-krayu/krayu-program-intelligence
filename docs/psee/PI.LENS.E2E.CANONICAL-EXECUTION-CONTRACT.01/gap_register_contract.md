## 5. Gap Remediation Rules

1. Every active gap with severity HIGH or MEDIUM requires a dedicated remediation contract before it can be closed.
2. Remediating a gap that changes pipeline behavior requires a parity validation proof (14-check or equivalent).
3. Closing a gap does not automatically trigger a new pipeline run — operator approval required.
4. This gap register must be updated whenever a new gap is discovered or an existing gap is closed.
5. A gap discovered during forensic analysis (R-stream contracts) must be registered here before remediation begins.
