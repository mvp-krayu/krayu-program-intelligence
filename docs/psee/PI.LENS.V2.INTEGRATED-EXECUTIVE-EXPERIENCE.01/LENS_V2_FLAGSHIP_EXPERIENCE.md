## Flagship Experience Philosophy

The LENS v2 Flagship Executive Intelligence Experience is not a report renderer. It is a command surface for structural operational intelligence.

The platform is built on a single strategic principle: **every visual decision must reinforce structural meaning**. Motion reinforces meaning. Density reinforces cognition. Emphasis reinforces authority. Nothing is decorative.

The experience is built to communicate:

- Evidence exists
- The evidence has been interpreted by a governed intelligence layer
- The result is structurally reliable
- Uncertainty is explicitly declared, not hidden
- The executive can trust what they see

## Cinematic Executive Intelligence Doctrine

Cinematic rendering is not aesthetic ambition. It is a deliberate governance choice.

The clinical, structured reveal sequence exists because executive decision-making requires cognitive preparation. Information delivered before the executive is cognitively ready is information that will be misread. The BLOCKED_ASSERTIVE profile reveals in 80ms because a blocked state must be impossible to miss. The AUTHORITATIVE profile reveals in 200ms with 80ms stagger because a confident result should feel grounded, not rushed.

Motion is governance data, not CSS values. Every timing constant is sourced from the MotionSemanticController.js governance layer and treated as an auditable artifact.

**Reveal sequence per state:**
- `EXECUTIVE_READY`: READINESS_BADGE → EXECUTIVE_NARRATIVE → PROPAGATION_POSTURE → EVIDENCE_POSTURE
- `EXECUTIVE_READY_WITH_QUALIFIER`: READINESS_BADGE → QUALIFIER_NOTICE → EXECUTIVE_NARRATIVE → PROPAGATION_POSTURE → EVIDENCE_POSTURE
- `DIAGNOSTIC_ONLY`: DIAGNOSTIC_ESCALATION → READINESS_BADGE → QUALIFIER_NOTICE → EXECUTIVE_NARRATIVE → PROPAGATION_POSTURE
- `BLOCKED`: BLOCKED_ESCALATION (sole phase)

The reveal sequence is deterministic. The same state always produces the same reveal. No randomization. No probabilistic transitions. No AI-mediated pacing.

## Operational Gravity Doctrine

Operational gravity is the visual weight system that establishes executive authority and evidence seriousness.

Each render state produces a distinct gravity token:
- `EXECUTIVE_READY` → `gravity-standard` — confident operational authority
- `EXECUTIVE_READY_WITH_QUALIFIER` → `gravity-qualifier` — qualified authority with qualifier emphasis
- `DIAGNOSTIC_ONLY` → `gravity-diagnostic` — advisory review weight

Operational gravity is not decoration. It is the first signal an executive receives that tells them the severity of what they are about to see.

The OperationalGravitySystem never gamifies intelligence, never creates AI-oracle effects, and never simulates autonomous cognition.

## Investigation Choreography Philosophy

Investigation in LENS v2 is bounded and sequential:

`SUMMARY → EVIDENCE → PROPAGATION → EXPLAINABILITY → LINEAGE`

This is not free-form exploration. It is a governed navigation model that:
1. Forces the executive to establish context before examining detail
2. Preserves the evidence-first doctrine at the UX layer
3. Prevents evidence from being presented out of causal sequence
4. Eliminates the possibility of arbitrary topology traversal

There is no text input in the investigation flow. There is no NLQ surface. There is no AI-mediated navigation. Navigation is deterministic: the structural sequence of the evidence is the only guide.

## Boardroom Presentation Philosophy

The ExecutiveBoardroomMode exists for one purpose: creating the conditions for governed intelligence to land with authority in a room.

The boardroom mode is not a slideshow. It is not a dashboard projected on a wall. It is a full-screen intelligence briefing that:
- Paces reveal to allow the room to absorb each structural finding
- Keeps qualifiers globally visible — they cannot be suppressed for aesthetic reasons
- Treats BLOCKED and DIAGNOSTIC states as non-negotiable — they cannot be softened for the room
- Eliminates all clutter, prompt surfaces, and non-structural elements

The pacing map enforces this:
- `EXECUTIVE_READY` → `CONTROLLED` pacing, `STAGED` reveal — fullscreen optimal
- `EXECUTIVE_READY_WITH_QUALIFIER` → `CONTROLLED` pacing, `STAGED_WITH_QUALIFIER_HOLD` — qualifier gets deliberate dwell time
- `DIAGNOSTIC_ONLY` → `ADVISORY` pacing — not fullscreen optimal
- `BLOCKED` → `HOLD` pacing — the room must see the blocked state in full

## Executive Trust Reinforcement Strategy

Trust in LENS v2 is built through explicit constraint:

1. **What you see is what was computed.** The rendering layer never supplements, infers, or rewrites evidence.
2. **Uncertainty is always declared.** Q-01..Q-03 qualifiers are globally visible across all density classes. They cannot be suppressed. Q-04 renders an explicit absence notice.

4. **Diagnostic states are assertive.** DIAGNOSTIC_ONLY escalation always appears before content.
5. **Evidence references are always preserved.** No density class removes evidence access.

The governance invariants embedded in the MotionSemanticController, IntelligenceDensityOrchestrator, and flagship orchestration layer are immutable. No downstream contract can suppress them.
