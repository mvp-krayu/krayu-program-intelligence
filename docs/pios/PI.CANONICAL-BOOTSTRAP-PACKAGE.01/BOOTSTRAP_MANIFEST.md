## Amnesia Recovery Test

### If a new Claude instance opens with only the repository:

**Phase 1 (CLAUDE.md auto-loads):** Instance knows execution rules, governance, evidence-first doctrine. Can operate correctly on any task. Does NOT know PI's commercial identity or cognition architecture.

**Phase 1 + Phase 2 (5 documents loaded):** Instance knows Runtime Cognition (6 primitives), Execution Blindness (synthesis layer), EIR dual-mode (activation contract), SW-Intel relationship (sibling). Can work on PI cognition tasks correctly.

**Phase 1-3 (9 documents loaded):** Instance knows the category (Execution Blindness), the engine (Measurement Boundary Discovery), the commercial offer (Discovery €15-25K), and the corridor framework (one analysis, five projections). Can work on commercial and product tasks correctly.

**Phase 1-4 (12 documents loaded):** Instance knows what mistakes to avoid, what patterns work, how ChatGPT and Claude collaborate. Can execute effectively without repeating costly errors.

**Phase 1-5 (all loaded):** Instance has full specimen context, validation criteria, and adapter strategy. Can work on StackStorm validation, new specimen onboarding, or customer engagement preparation.

**Recovery time:** ~45 minutes of document reading for full Phase 1-4. Approximately 80% of current operational capability.

**What cannot be recovered from repository alone:** The experience of discovering these things. The emotional weight of Kurt saying "STOP — do not accept model behavior as root cause." The iterative process that turned "the model is nondeterministic" into "the context was wrong." These are encoded in the forensics document but cannot reproduce the judgment that comes from having lived through the failure loop.

## Manifest Governance

This manifest is itself a governed artifact. It should be updated when:

1. A new CONSTITUTIONAL document is created
2. A HYPOTHESIS is promoted to BOOTSTRAP (e.g., StackStorm passes pipeline validation)
3. A COMMERCIAL document is created or superseded
4. The loading sequence changes based on experience

It should NOT be updated for:
- Individual specimen changes
- Code-level implementation changes
- Session-specific findings that don't survive the "would this be lost?" test
