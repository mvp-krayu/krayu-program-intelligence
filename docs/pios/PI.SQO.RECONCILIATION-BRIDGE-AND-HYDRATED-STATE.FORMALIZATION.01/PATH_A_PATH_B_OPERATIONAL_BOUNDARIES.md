# PATH A / PATH B Operational Boundaries

> **Permanent separation of concerns. PATH A is frozen substrate. PATH B is operational intelligence. Neither absorbs the other.**

---

## 1. The Boundary

PATH A and PATH B are complementary, not competing. This document makes their operational boundaries permanent and explicit.

```
PATH A                              PATH B
(Structural Grounding)              (Semantic Reconstruction)
─────────────────────               ─────────────────────────
Proves claims                       Produces intelligence
Deterministic                       Operationally useful
Vault-anchored                      Semantically coherent
Certified baseline                  Renders with disclosure
FROZEN                              ACTIVE
```

---

## 2. PATH A: Frozen Deterministic Substrate

### Status
OPERATIONAL but FROZEN as certified baseline since e8a369d (2026-05-08).

### What PATH A Owns
- Vault anchors (structural evidence backing for semantic claims)
- Topology reports (observational structural evidence)
- CLU/DOM/CAP registries (structural entity registries)
- Grounding ratio computation (per-domain grounding measurement)
- DPSIG signal family (deterministic signal derivation from structure)

### What PATH A Does
- Provides structural proof for semantic claims
- Produces grounding ratios that feed Q-class computation
- Supplies vault anchors for reconciliation bridge correspondence
- Remains the deterministic substrate against which semantic claims are tested

### What PATH A Does NOT Do
- Interpret semantic meaning
- Render executive intelligence
- Manage qualification state
- Extract evidence from source documents
- Reconstruct semantic intent

### PATH A Is Not Being Reopened

This is explicit architectural doctrine:

**PATH A's philosophy, architecture, and certified baseline are not subject to redesign.** The DPSIG signal family, vault anchoring model, and topology verification approach are validated and frozen. Future work extends PATH A's coverage (more vault anchors, more topology data) but does not change its architecture.

Reopening PATH A means:
- Questioning deterministic grounding (not productive)
- Redesigning signal derivation (not needed — commercially validated)
- Rearchitecting vault anchors (not warranted — model works)

None of these are in scope for any near-term stream.

## 4. The Relationship

```
PATH A provides: structural proof
PATH B provides: operational intelligence

PATH A feeds: reconciliation bridge (structural evidence input)
PATH B feeds: reconciliation bridge (semantic domain input)

Neither owns: the reconciliation bridge
Neither replaces: the other
```

### What Connects Them

The **reconciliation bridge** is the only architectural component that operates across both paths. It takes PATH B's semantic domain model and PATH A's structural registries and compiles correspondence. Neither path needs to know about the other's internals.

### What Does NOT Connect Them

- PATH B does not read PATH A's internal structures directly
- PATH A does not interpret PATH B's semantic claims
- No runtime component straddles both paths
- No extraction pipeline serves both paths simultaneously

## 6. Anti-Collapse Doctrine

### Semantic Richness Must Not Collapse

The system's most commercially valuable asset is its semantic richness — the ability to produce executive-grade operational intelligence from evidence. The following directions are explicitly rejected as primary semantic strategy:

- **Regex/token harvesting** as replacement for semantic reconstruction
- **Pattern matching** as replacement for domain model hydration
- **Keyword extraction** as replacement for actor-based semantic analysis
- **Statistical correlation** as replacement for crosswalk correspondence

These techniques may be tools within the pipeline. They are not the pipeline.

### PATH A Must Not Absorb PATH B

If PATH A expanded to include semantic interpretation, the system would lose:
- Honest disclosure (PATH A is binary; HYDRATED requires graduated trustworthiness)
- Commercial velocity (PATH A requires structural proof before rendering)
- Semantic richness (PATH A's strength is determinism, not interpretation)

### PATH B Must Not Absorb PATH A

If PATH B expanded to include structural verification, the system would lose:
- Deterministic certainty (PATH A's verification is binary and certified)
- Certified baseline (PATH A's frozen state is a governance asset)
- Separation of concerns (the reconciliation bridge exists precisely to mediate)
