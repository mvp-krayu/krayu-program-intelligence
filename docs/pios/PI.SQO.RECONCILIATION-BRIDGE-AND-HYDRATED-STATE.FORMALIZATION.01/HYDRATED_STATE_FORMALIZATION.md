# HYDRATED State Formalization

> **Operational boundaries, advancement criteria, and what HYDRATED protects.**

---

## 1. HYDRATED Definition (Locked)

HYDRATED is already CANONICAL in TERMINOLOGY_LOCK.md:

> The state where semantic reconstruction has achieved high internal coherence and operational usefulness, but without complete structural grounding proof for every semantic claim.

This document does not redefine HYDRATED. It formalizes its **operational boundaries** — what HYDRATED includes, what it excludes, how it advances, and what it protects.

A client at HYDRATED has:

| Capability | Evidence |

HYDRATED is a complete operational state. It delivers executive intelligence. It is commercially valuable.

## 3. What HYDRATED Excludes

A client at HYDRATED does NOT have:

| Missing | Why | Path to Acquire |

| Complete structural grounding | Not all semantic claims have vault anchors | PATH A progressive grounding |

| Authority promotion | Governance certification requires EXACT or higher | Full trustworthiness progression |

## 4. HYDRATED Is Not Degraded

This is a formal architectural doctrine:

**HYDRATED output is legitimate.** It is not waiting for "real" grounding. It is not a staging area. It is not incomplete in the sense that it should be hidden or discounted.

The rationale:
- Semantic coherence from multiple corroborating sources has independent operational value
- Executive intelligence with honest disclosure is commercially superior to no intelligence pending proof
- Blocking rendering on structural proof destroys the system's primary value proposition
- Q-02 governance (mandatory disclosure) makes the trustworthiness level explicit

## 5. HYDRATED Advancement Criteria

### HYDRATED → RECONCILED

The transition from HYDRATED to RECONCILED requires:

| Criterion | Requirement |
|---|---|
| Correspondence compiled | Reconciliation bridge has processed all 17 domain clusters |
| Per-domain confidence assigned | Each domain has a graduated confidence level (1-5) |
| Unmatched claims identified | Semantic claims without structural evidence are explicitly listed |
| Unmatched evidence identified | Structural evidence without semantic consumers is explicitly listed |
| Human review completed | Operator has reviewed and approved the correspondence report |
| No blocking contradictions | No domain where structural evidence contradicts semantic claim |

The transition is NOT automatic. It requires:
1. The reconciliation bridge to compile correspondence
2. A human operator to review and approve
3. SQO to record the state transition

### What Triggers Reconciliation

Reconciliation is not a background process. It is triggered by:
- Operator decision to advance a client toward higher trustworthiness
- New structural evidence becoming available (vault anchors, topology reports)
- AI-assisted semantic reconstruction producing new correspondence candidates
- Governance stream requesting reconciliation audit

## 6. HYDRATED Protection Doctrine

### What HYDRATED Protects Against

| Threat | How HYDRATED Protects |
|---|---|
| Semantic richness collapse into regex harvesting | HYDRATED legitimizes semantic reconstruction as operational output |
| PATH A philosophical reopening | HYDRATED does not require PATH A to change — it uses PATH A as-is |
| Rendering blockage pending proof | HYDRATED renders now with disclosure, not later with proof |
| Repeated rediscovery of semantic intent | HYDRATED formalizes that semantic work is done and valuable |
| Binary grounding pressure | HYDRATED introduces graduated trustworthiness, not binary pass/fail |

### What Cannot Degrade HYDRATED

- New PATH A requirements do not invalidate HYDRATED output
- Failed reconciliation attempts do not downgrade HYDRATED status
- Additional structural evidence does not retroactively disqualify HYDRATED rendering
- Future AI-assisted reconstruction extends HYDRATED, does not replace it
