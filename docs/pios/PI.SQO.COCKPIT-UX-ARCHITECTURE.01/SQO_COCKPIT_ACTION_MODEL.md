# SQO Cockpit Action Model

PI.SQO.COCKPIT-UX-ARCHITECTURE.01

---

## Allowed Actions

### Inspect debt
- View semantic debt inventory
- Filter by category, severity, blocking status
- Sort by priority score
- Drill down into individual debt items
- View remediation pathway per item

### Inspect evidence
- View evidence chains from source artifacts to derived metrics
- Verify input hash integrity
- Trace provenance from qualification score to originating artifact field
- View source commit reference

### Inspect continuity
- View continuity assessment metrics
- Identify coverage gaps (structural nodes without semantic mapping)
- View label fidelity breakdown
- Identify lineage gaps per domain

### Inspect maturity
- View 8-dimension maturity profile
- Drill down per dimension (D1-D8) to see formula inputs
- View semantic gravity assessment
- View qualification stability assessment
- Compare dimension scores across clients (if multiple registered)

### Inspect progression readiness
- View current S-state and target
- View readiness score
- View blocking debt items
- View required remediation pathways
- View S-state success criteria

### Mark source material requirement as requested
- For a specific debt item or remediation pathway
- Updates cockpit-level tracking status: IDENTIFIED → REQUESTED
- Does NOT modify any SQO artifact
- Additive tracking record only

### Prepare re-run checklist
- Generate a checklist of prerequisites for semantic pipeline re-run
- Based on current debt inventory and remediation pathways
- Includes source material verification, pathway readiness, governance checks
- Exported as a planning artifact (does not trigger re-run)

### Prepare handoff package
- Assemble qualification evidence for PATH B submission
- Validates minimum criteria (S2+, certifications passing, replays passing)
- Produces a package summary with audit trail
- Does NOT emit into LENS

### Compare qualification snapshots
- Compare two historical qualification states for the same client/run
- Show delta in S-state, maturity scores, debt counts
- Identify what changed between assessments
- Requires qualification_history artifact

### Export governance packet
- Export a governance compliance summary
- Includes certification statuses, replay verification results, provenance chains
- For audit and compliance purposes

## Forbidden Actions

### Edit source artifacts
SQO Cockpit is read-only against all source artifacts (Lane A, Lane D, semantic pipeline outputs). No modification of source data.

### Edit debt items
Debt items are deterministically generated by the debt detection engine. Manual editing would break determinism and replay-safety.

### Override S-state
S-state is deterministically detected by the qualification state engine. No manual override. S-state changes only through artifact evidence changes.

### Override Q-class
Q-class is a deterministic pure function of qualifier resolver. No override from the cockpit.

### Fabricate source material
Source material must come from the client. No AI-generated documentation, no synthetic business labels, no fabricated ADRs.

### Auto-generate semantic labels
Domain labels must derive from client-provided source material through the semantic pipeline. No cockpit-side label generation.

### Approve projection directly
Projection authorization is PATH B's authority. The cockpit prepares handoff packages — it does not approve projection.

### Push data into LENS directly
The cockpit never emits into the LENS projection surface. All LENS rendering goes through PATH B.

### Bypass PATH B
No shortcut from SQO Cockpit to LENS. The handoff model is mandatory.
