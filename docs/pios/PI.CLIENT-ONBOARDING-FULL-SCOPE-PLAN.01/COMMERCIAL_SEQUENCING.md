The full-scope plan's roadmap (Phase 0→1→2…) is an **engineering** sequence. The operative sequence is **commercial**, by risk profile, not build order. Scope is unchanged; only order changes.

## The governing question

Not "is the portal built?" but: **Can I reliably take repo → cognition → assessment → advisory for a new client?** Everything else is optimization.

## The dominant risk

Not insufficient UX. **Insufficient proof of repeatability.**

## Three Tracks

| Track | What | When |
|---|---|---|
| **A — Factory Certification** | Phase 0: fresh raw ingestion proven (NetBox from tar, no artifact reuse). The single most dangerous UNVERIFIED row. Until proven, everything is hypothetical. | **NOW — mandatory, blocking** |
| **B — Operator-Led (Concierge) Commercialization** | Client never sees a portal. They: book assessment → sign NDA → provide repo → receive report → attend walkthrough. Operator drives the contract by hand. **Sellable before any portal exists.** | After A |
| **C — Productization** | Portal, cockpit, token flow, lifecycle UI, automation, event bus. | Only after A + B |

## The steer for the next discussion

Not "how do I build the portal?" but: **"How do I deliver the first 5 client assessments using operator-led onboarding only?"**

If 5 clients can be onboarded manually, the portal is an optimization problem. If they cannot, the portal is a distraction.

## Disposition

- Plan: **accepted**.

- Immediate action: **run Phase 0**.
- Next discussion: concierge delivery of the first 5 assessments — not portal construction.
