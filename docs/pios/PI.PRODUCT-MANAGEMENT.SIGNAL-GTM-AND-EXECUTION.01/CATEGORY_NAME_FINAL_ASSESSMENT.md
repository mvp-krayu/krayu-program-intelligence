| Dimension | Assessment | Score |

| **Memorability** | Two words. Clean. "Structural intelligence" — sticks immediately. Easy to Google. Easy to reference in an email subject. | 9/10 |

| **Expansion potential** | High. "Structural" encompasses: software structure, organizational structure, portfolio structure, M&A target structure. ## Decision

### **Structural Intelligence** — LOCKED as product category name

```
COMMERCIAL WEDGE          Software Intelligence
                          (first domain module, first exploit)

PRODUCT CATEGORY          Structural Intelligence
                          (what the product delivers)

PLATFORM CATEGORY         Program Intelligence
                          (the discipline, the destination)
```

**Why "Structural Intelligence" wins:**

1. **Two words beat three.** CTOs repeat two-word categories. They shorten three-word categories. If the natural abbreviation IS the name, use the abbreviation.

2. **"Structural" is the differentiator.** No competitor occupies "structural." SonarQube = code quality. CodeScene = behavioral code analysis. Jira = work management. "Structural" is unclaimed territory.

3. **Expansion is natural.** Software Intelligence → Structural Intelligence about software. M&A Intelligence → Structural Intelligence about acquisitions. Portfolio Intelligence → Structural Intelligence about portfolios. The product category doesn't change when domain modules are added.

4. **The ambiguity is a feature.** "Structural Intelligence" invites the question: "Structural intelligence about what?" Answer: "About how your program actually executes. The hidden topology, the propagation risks, the coupling dynamics." The question IS the sales conversation opener.

**How it flows in practice:**

> "What does Signäl do?"
>
> "Structural Intelligence. We reveal the structural reality of how your program executes — fragility, coupling inertia, dependency amplification. Things no other tool can show you."
>
> "How?"
>
> "Software Intelligence is our first module — it derives structural consequences from your code architecture. But the platform is Program Intelligence — governed structural assessment across any execution domain."

**Update to SKU model:** Replace "Structural Execution Intelligence" with "Structural Intelligence" in all references. The hierarchy becomes:

- Software Intelligence (wedge / first domain module)
- Structural Intelligence (product category)
- Program Intelligence (platform / discipline)
