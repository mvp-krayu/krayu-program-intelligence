# Operational UX Doctrine

## Principle

The SQO Cockpit communicates operational qualification state through visual hierarchy,
not through equal-weight panel scanning. The operator comprehends:

1. Current state — within seconds
2. What blocks progression — immediately visible
3. What happens next — guided by workflow spine
4. What debt matters later — collapsed, accessible on demand

## Doctrine Rules

1. **Attention hierarchy is mandatory.** Every panel has a prominence level. No two zones compete equally.
2. **Blockers dominate.** Critical blockers overpower all secondary content.
3. **Deferred debt is collapsed.** Expansion debt for future S-states does not pollute primary cognition.
4. **Workflow spine is persistent.** Operator always sees the remediation pathway.
5. **Visual state derives from SQO state.** No client-name branching. S1 is amber. S2 is blue. S3 is green.
6. **Progressive disclosure is deterministic.** Same journey state always produces same visibility.
7. **No AI orchestration.** All attention, grouping, and visibility rules are deterministic.
8. **Forensic detail is available but subordinate.** Full debt explorer, evidence, maturity internals accessible through section navigation.
