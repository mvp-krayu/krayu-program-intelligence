## Primary Interaction Model

**The primary interaction model for Program Intelligence is structural exploration, not conversational querying.**

The topology graph, evidence lineage tracer, and domain deep-dive are the center of gravity. The user investigates by navigating structure: clicking domains, traversing edges, tracing lineage, expanding evidence chains, observing temporal drift. Query (Phase 5B.2) is secondary — it adds explanation on top of an already-powerful structural exploration instrument.

This is the core product differentiator. The topology + evidence traversal system is the innovation.

---

## Governance Concept: INVESTIGATIVE AUTHORITY

The current governance model has a binary split:

```
DETERMINISTIC (rendering) <--> INTERPRETIVE (75.x authorized synthesis)
```

This roadmap introduces a missing middle layer:

```
DETERMINISTIC --> INVESTIGATIVE --> INTERPRETIVE
     |               |               |
  rendering    guided traversal   bounded synthesis
  no selection  topology-derived   evidence-bound
  pure display  structural adjacency  75.x required
```

**Investigative Authority** permits:
- Surfacing connected domains based on topology edges
- Exposing unresolved neighboring pressure zones via structural adjacency
- Traversing evidence lineage through propagation chains
- Showing structural adjacency and cluster membership
- Presenting temporal deltas between measured epochs
- Navigating qualification blockage via debt linkage

**Investigative Authority is NOT:**
- Inference (no "this probably means...")
- Recommendation (no "you should look at...")
- Prioritization (no "this matters more than...")
- Prediction (no "this will likely...")

**Governing principle:** Every navigation path is derivable from payload topology. No path exists that requires inference to construct. Selection criteria are topology-derived, not inference-derived.

**Governance classification:** L6/L7 consumption. No 75.x authorization required. No CLAUDE.md amendment needed.

### Phase 5A — Guided Structural Investigation

**Authority:** INVESTIGATIVE (topology-derived guided traversal)

**UX identity:** Structural excavation. Evidence traversal. Investigative navigation. Progressive contextualization. Guided operational cognition.

**NOT:** Chatbot. Copilot. Assistant. Recommendation engine.

#### 5A.1 — Domain Structural Deep-Dive

Click any domain to open structural investigation panel exposing the full evidence chain behind grounding state.

**Data:** reconciliation_correspondence.enriched.v1.json (evidence_factors, crosswalk_lineage_status, structural_component_count, structural_evidence_refs, correspondence_basis, enrichment_metadata)

**Sections:** Domain Identity | Evidence Chain | Grounding State

**Files:** IntelligenceField.jsx, StructuralTopologyZone.jsx, GenericSemanticPayloadResolver.js, flagshipBinding.js, lens-v2-flagship.js, lens-payload.js

#### 5A.2 — Evidence Lineage Tracer

From any signal, trace backward: Signal -> Pressure Zone -> Origin Domain -> Structural Evidence -> Source Path.

**Data:** semantic_evidence_intake.v1.json (source_class, source_path, hash_verified, eligible_operations), evidence_blocks, propagation_summary

**Files:** IntelligenceField.jsx, flagshipBinding.js, lens-v2-flagship.js

#### 5A.3 — Structural Blockage Visibility

What structurally prevents S-state advancement. Pure structural exposure.

**Data:** runtime_qualification_projection.v1.json, semantic_debt_inventory.v1.json, progression_readiness.v1.json, maturity_dimension_breakdown.v1.json

**EXPLICIT PROHIBITIONS:**
- NO remediation prioritization
- NO ranked next actions
- NO optimization advice
- NO intervention recommendation
- NO "best path" or "recommended approach"
- NO operational prescription
- NO "you should" language
- NO implied urgency ordering

**MAY expose:** What blocks (structural fact). Which debts exist (structural inventory). Which evidence gaps remain (structural absence). Dimension scores (structural measurement).

**MUST NOT imply:** What leadership should do. Which blockage is most important. What order to address them in. Whether the blockage matters.

The system shows the wall. It does not point at the door.

**Files:** IntelligenceField.jsx, flagshipBinding.js, lens-v2-flagship.js

#### 5A.5 — Temporal Structural Intelligence

**THE ARCHITECTURAL TRANSITION POINT.**

Without temporal cognition: advanced structural assessment explorer (static).
With temporal cognition: live operational intelligence environment.

This sub-phase bridges LENS v2 from assessment product to operational intelligence system.

**Data:** reconciliation_temporal_analytics.v1.json (epoch_timeline, level_transitions, persistent_domains, resolved_domains, enrichment_effectiveness, structural_semantic_divergence, degradation.signals), reconciliation_lifecycle.v1.json (per_domain records, epochs)

**Surfaces:** Structural drift. Pressure evolution. Qualification trajectory. Domain movement. Persistent gaps. Divergence indicators.

**Does NOT do:** Trend extrapolation. Projections. Forecasting. "Improving"/"worsening" judgments. Shows: level was X at T1, now Y at T2. The delta is the fact.

**Files:** TemporalIntelligenceView.jsx (new) or IntelligenceField.jsx, flagshipBinding.js, lens-v2-flagship.js

#### 5A.6 — Integrated Investigation Flow

Integration layer connecting 5A.1-5A.5 into a coherent structural investigation session.

Breadcrumb navigation. Cross-surface traversal. Investigation state. Back navigation. Structurally-derivable navigation options at each point.

Ships last because it must integrate temporal navigation alongside domain, lineage, and blockage traversal.

**Files:** IntelligenceField.jsx, all 5A components
