# CHRONICLE GENESIS ADAPTATION

> **Status:** LOCKED — How the Chronicle evolves from semantic replay documentation into onboarding-origin cognition traversal.

---

## 1. The Adaptation

The BlueEdge chronicle (RC-01 through RC-09) proved the chronicle model on post-genesis semantic replay. That model now adapts to capture cognitive genesis — cognition forming from raw intake.

This is not a redesign. It is an extension of proven architecture into the upstream origin layer.

---

## 2. What Changes

### 2.1 Chronicle Starts Earlier

**Before (Semantic Replay):** Chronicle begins at checkpoint_00_baseline — artifacts already exist. The system replays governance over materialized substrates.

**After (Cognitive Genesis):** Chronicle begins at DISCOVERY — raw evidence encounter. The system captures cognition forming from nothing.

### 2.2 New Semantic Phase: DISCOVERY

The 8-phase semantic rhythm gains a genesis-native predecessor:

```
DISCOVERY       ← NEW: raw intake, source boundary, evidence acquisition
EMERGENCE       — first structural topology materializes (was first phase)
FORMATION       — semantic claims crystallize
TENSION         — governance friction
STRENGTHENING   — evidence enrichment
STABILIZATION   — deterministic replay confirms
QUALIFICATION   — state transition earned
CONVERGENCE     — cross-specimen observation
PROJECTION      — cognition communicable
```

### 2.3 Chronicle Events Are Runtime-Native

**Before:** Chronicle artifacts created as governance stream outputs. The chronicle is compiled post-hoc from checkpoint/spine/governance data.

**After:** Chronicle events emit inline during pipeline execution. The chronicle accumulates as cognition forms. Compilation renders the accumulated events — it does not create them.

### 2.4 Hero Moments Are Chronicle-Native

**Before:** Hero moments documented in separate inventory files. Referenced by chronicle but not structurally integrated.

**After:** Hero moments are first-class chronicle objects (`chronicle_hero_moment`). They are encountered during traversal — cognitive discontinuities in the narrative flow.

### 2.5 AI Assistance Is Chronicle-Native

**Before:** AI assistance happens outside the chronicle. Results may be referenced.

**After:** AI interventions are first-class chronicle objects (`chronicle_ai_intervention`). Every AI suggestion, detection, and explanation is traced in the cognitive lineage.

### 2.6 Learning Events Are Chronicle-Native

**Before:** Learning events captured in separate JSONL files. Lifecycle managed outside chronicle.

**After:** Learning events are first-class chronicle objects (`chronicle_learning_event`). The chronicle traces how onboarding experience becomes governed learning.

## 5. Event Density Model

Not every genesis phase produces equal chronicle density:

| Phase | Expected Event Density | Reason |
|-------|----------------------|--------|
| DISCOVERY | LOW — 3-5 events | Intake is usually clean. High density signals problems. |
| EMERGENCE | MEDIUM — 5-10 events | Structural topology reveals structural reality. Hero moments possible. |
| FORMATION | MEDIUM-HIGH — 10-20 events | Proposition derivation produces many claims. |
| TENSION | HIGH — 20+ events | Governance review produces many events. This is where friction lives. |
| STRENGTHENING | MEDIUM — 5-10 events | Enrichment is focused. |
| STABILIZATION | LOW — 3-5 events | Revalidation is deterministic. |
| QUALIFICATION | LOW — 1-3 events | State transition is a governed moment. |
| CONVERGENCE | MEDIUM — 5-10 events | Cross-specimen comparison (when applicable). |
| PROJECTION | LOW — 1-3 events | Chronicle compilation and certification. |

If DISCOVERY is HIGH density, the chronicle has found an interesting intake. If TENSION is LOW density, governance had nothing to challenge.

## 6. Migration Path

### 6.1 Existing BlueEdge Chronicle

No changes. The BlueEdge chronicle remains REPLAY-CERTIFIED as a post-genesis semantic replay corridor. Its checkpoint/chapter/zoom model is the proven reference implementation.

### 6.2 Future Genesis Chronicles

New onboardings that capture genesis will use the extended model:
- Chapter 0 (DISCOVERY) added
- Chronicle events emitted inline
- Hero moments, AI interventions, learning events as chronicle-native objects
- Checkpoint model extended to cover DISCOVERY phase

### 6.3 NetBox Retrospective

NetBox onboarding was the first informal genesis specimen. A retrospective genesis chronicle could be composed from existing artifacts (pipeline output, hero moment inventory, governance events, learning events) — but this is optional. The forward model matters more than retroactive capture.
