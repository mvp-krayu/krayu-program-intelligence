# GENESIS REPLAY DOCTRINE

> **Status:** LOCKED — Canonical architectural doctrine for onboarding-origin cognitive replay.

---

## 1. Definition

**GENESIS** is the governed onboarding-origin cognition layer of Program Intelligence.

It captures how client understanding **begins** — from raw intake, through structural and semantic cognition, to governed projection — as a replayable, evidence-first model.

GENESIS is not a product, not a module, not a tool. It is the **cognitive origin layer** — the governed answer to: *Where does cognition begin?*

---

## 2. The Two Replay Corridors

Program Intelligence recognizes two distinct replay corridors:

### 2.1 GOVERNED SEMANTIC REPLAY (Proven)

Replay over already-materialized semantic substrates.

**Operates on:**
- SDC outputs (candidate CSR, derivation reports)
- CSR domains (semantic ontology)
- Vault claims (structural evidence backing)
- Topology artifacts (40.x structural models)
- Governance events (operator review, arbitration)

**Proves:** That governance lifecycle patterns transfer across specimens. That operator review, enrichment, revalidation, and qualification work on post-genesis artifacts.

**Does NOT prove:** How the artifacts came to exist. How raw intake becomes structured cognition. Where structural surprises first emerge.

**Reference specimen:** BlueEdge Governed Cognitive Replay Chronicle (RC-01 through RC-09, REPLAY-CERTIFIED).

### 2.2 FULL COGNITIVE GENESIS REPLAY (This Doctrine)

Replay from raw onboarding intake — the full cognitive origin story.

**Operates on:**
- Raw source archives (GitHub repos, evidence documents)
- Source boundary verification (SHA256, intake manifests)
- Artifact normalization (extraction, structural scanning)
- First topology emergence (40.2 → 40.3 → 40.4)
- First Hero Moment emergence (structural surprises at PATH A)
- Semantic formation (SDC/SPE derivation)
- Governance friction (operator review, contested claims)
- Substrate strengthening (enrichment, reconciliation)
- Qualification progression (S0 → S1 → S2)
- Projection (LENS, SQO, Chronicle)

**Proves:** How Program Intelligence cognition is born from raw evidence. The full chain from "unknown client" to "governed structural understanding."

**Critical distinction:** Governed semantic replay validates the governance lifecycle on existing artifacts. Cognitive genesis replay captures the full cognitive origin — from nothing to understanding.

## 3. Where Cognition Begins

Cognition begins at **evidence boundary establishment** — the moment the system first encounters raw client evidence.

The genesis chain:

```
RAW INTAKE (source archive + evidence files)
  → evidence/source discovery (what exists?)
  → intake normalization (what structure does it have?)
  → PATH A structural formation (what is the topology?)
  → first Hero Moments (what is surprising?)
  → PATH B semantic formation (what does it mean?)
  → spine fusion (how do structural and semantic connect?)
  → SQO governance (does the operator accept this understanding?)
  → learning events (what did we learn from this client?)
  → substrate strengthening (can weak claims be enriched?)
  → replay validation (does it all hold deterministically?)
  → LENS/SQO/Chronicle projection (can cognition be communicated?)
```

The chronicle must not start after cognition already exists. The chronicle must observe cognition forming.

## 5. Genesis Replay Requirements

For a future onboarding to be captured as a genesis replay:

1. **Every phase must emit a checkpoint artifact** — frozen state at each genesis boundary
2. **Hero Moment candidates must be surfaced inline** — not retrospectively
3. **AI-assisted actions must be logged as discrete events** — input, suggestion, operator decision
4. **Learning events must be captured at point of discovery** — not batched post-hoc
5. **The replay must be deterministic** — same inputs + same registry state → same outputs
6. **The chronicle must be composable from genesis checkpoints** — not manually authored
