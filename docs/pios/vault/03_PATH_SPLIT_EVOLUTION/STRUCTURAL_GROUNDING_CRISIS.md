# Structural Grounding Crisis

> **The problem that created PATH A and PATH B — the system's most important architectural pivot.**

---

## The Crisis

By early May 2026, the system had:
- Rich semantic reconstruction (15-actor model, PIE vault heritage, domain intelligence)
- Structural infrastructure (vault anchors, topology reports, CLU/DOM/CAP registries)
- No clear model for how these related

The result was **architectural blur** — three overlapping confusions:

### Confusion 1: "Is semantic reconstruction valid without structural proof?"

The system's strongest asset — rich, internally-coherent semantic reconstruction from evidence — was being treated as "ungrounded" and potentially illegitimate. This created pressure to either:
- Block rendering until structural proof existed (destroying operational value)
- Pretend structural proof existed (fabricating governance compliance)

### Confusion 2: "Are extraction and grounding the same thing?"

Evidence extraction (pulling semantic candidates from HTML) and structural grounding (proving those candidates have vault-backed structural correspondence) were partially conflated. The same modules were handling both concerns.

### Confusion 3: "What does 'qualification' mean?"

The system had no vocabulary for graduated trustworthiness. A semantic claim was either:
- Grounded (trusted)
- Not grounded (suspect)

There was no middle ground — no way to say "this is semantically coherent and operationally useful but not structurally proven."

## What Failed

The binary compliance model (grounded/not grounded) from the snapshot era:
- Could not express partial confidence
- Could not serve executive intelligence without structural proof
- Could not distinguish semantic coherence from structural verification
- Treated HYDRATED output as deficient rather than legitimate

## What This Produced

The structural grounding crisis produced two responses:

1. **[[PATH_A_EMERGENCE]]** — formalize structural grounding as a separate concern
2. **[[PATH_B_EMERGENCE]]** — formalize semantic reconstruction as a legitimate state (HYDRATED)

These paths are **complementary, not competing.** PATH A provides structural backing. PATH B provides semantic richness. The system delivers value at PATH B (HYDRATED) while progressively achieving PATH A (structural grounding).
