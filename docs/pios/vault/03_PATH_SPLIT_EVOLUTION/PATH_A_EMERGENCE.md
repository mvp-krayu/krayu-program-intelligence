# PATH A Emergence

> **Structural grounding — vault anchors, topology verification, deterministic evidence.**

---

## What PATH A Is

PATH A is the **structural grounding path** — the concern of proving that semantic claims have structural backing in vault anchors, topology reports, or verified registries.

## Why It Emerged

The DPSIG signal family work (2026-05-07) was the first time "PATH A" appeared as named terminology:

```
196fcd8  2026-05-07  [DPSIG] LENS Executive Validation —
                     "Path A deterministic intelligence commercially validated"
```

The DPSIG work proved that deterministic signal derivation from structural evidence could produce commercially valuable executive intelligence. This was "PATH A" — intelligence grounded in structural proof.

## What PATH A Contains

| Component | Layer | Role |
|---|---|---|
| Vault anchors | L3-L4 | Structural evidence backing for semantic claims |
| Topology reports | L2-L3 | Observational structural evidence |
| CLU/DOM/CAP registries | L3 | Structural entity registries |
| Grounding ratio computation | L4 | Per-domain grounding measurement |
| DPSIG signal family | L3-L4 | Deterministic signal derivation from structure |

## What PATH A Does NOT Provide

- Semantic interpretation (that's PATH B)
- Executive intelligence rendering (that's LENS v2)
- Qualification assessment (that's SQO)
- Evidence extraction (that's the extraction corridor)

PATH A provides **structural proof.** It does not provide **operational value** — that requires PATH B's semantic reconstruction.

## Graduated Grounding Model

The strategic roadmap defines a graduated model (replacing binary grounding):

| Level | Name | Evidence |
|---|---|---|
| 5 | STRUCTURALLY_GROUNDED | Vault-backed |
| 4 | OBSERVATIONALLY_CORROBORATED | Topology/crosswalk confirmed |
| 3 | SEMANTICALLY_COHERENT | Multiple semantic sources agree |
| 2 | UPSTREAM_EVIDENCE_BOUND | Operator evidence only |
| 1 | UNMAPPED | No binding |

**OPERATIONAL but PARTIAL.**
