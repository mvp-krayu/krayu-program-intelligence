## 8. Method

Each reference deep-dive follows a consistent structure:

- Why this reference (positioning)
- Core qualities to study
- What LENS borrows
- What LENS must avoid
- Typography read
- Color read
- Motion read (where applicable)
- Composition / chrome / depth read (where applicable)
- Inspection cues for LENS review passes
- Summary borrow phrase

This consistency allows the comparison matrix (`CINEMATIC_REFERENCE_COMPARISON_MATRIX.md`) to cross-tab borrows and avoids cleanly across the nine families.

The Playwright framework, comparison matrix, and dashboard syndrome detection guide were composed to operationalize the rubric defined in the companion stream's `VISUAL_EVALUATION_RUBRIC.md`. Mechanical signal definitions in the framework correspond directly to the rubric's six axes; detection thresholds in the syndrome guide correspond to the anti-dashboard rules in the companion stream.

Cross-checks performed:
- Alignment with `VISUAL_DIRECTION_DOCTRINE.md` — every deep-dive cites the doctrine clauses it serves.
- Anti-dashboard consistency — `DASHBOARD_SYNDROME_DETECTION_GUIDE.md` operationalizes the four primary tests in `ANTI_DASHBOARD_ENFORCEMENT.md` §6–§9 with no drift.
- Executive-cognition consistency — every deep-dive's "what LENS avoids" clause is grounded in `EXECUTIVE_COGNITION_MODEL.md` reader requirements.
- Playwright inspection feasibility — every mechanical signal in the framework is computable from a rendered DOM + screenshots; no signal requires speculative analysis.
- Cinematic realism coherence — the comparison matrix demonstrates LENS lives in the *intersection* of the reference set, not at any single reference; this is consistent with the doctrine's positioning.
- Operational seriousness preservation — every borrow clause supports operational seriousness; every avoid clause prevents drift away from it.
