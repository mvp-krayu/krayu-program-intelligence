## 1. What Is a Cognition Object?

A Cognition Object is a structured L4 artifact that produces novel understanding about a specimen's structural execution reality from governed pipeline inputs — independently of any audience, format, or rendering decision.

### 1.1 Constitutional Definition

> A Cognition Object is a deterministic, evidence-bound, audience-independent, projection-free structured artifact that answers a distinct cognitive question about a specimen's structural reality. It is produced by a single materializer within the PICR from CIP inputs, carries ZERO interpretive authority, and exists before any projection decision occurs.

### 1.2 Essential Properties

Every Cognition Object MUST exhibit ALL of the following properties:

| Property | Requirement | Rationale |
|----------|-------------|-----------|
| **Deterministic** | Same CIP → same object. No stochastic behavior. | Replayability is constitutional — non-deterministic output is not cognition, it is hallucination |
| **Evidence-bound** | Every field traces to a governed source artifact at L0-L3 | A field without provenance is an assertion, not cognition |
| **Audience-independent** | Contains no audience assumptions, no format decisions, no tone calibration | Audience awareness belongs to L5 (PRE). An object that changes based on who reads it is a projection, not cognition |
| **Projection-free** | Contains no rendering vocabulary, no narrative framing, no competitive positioning | Rendering belongs to L5. An object that assumes a delivery surface is a projection artifact |
| **Structurally novel** | Produces understanding that no other cognition object produces | Redundant objects are derived views, not cognition |
| **Single-question** | Answers one distinct cognitive question about the specimen | An object that answers multiple unrelated questions should be decomposed |
| **Zero authority** | Requires ZERO interpretive authority to produce | Any object requiring 75.x is a projection artifact, not a cognition object |

### 1.3 The Cognitive Question Test

Every Cognition Object answers exactly one question of the form:

> "What does Program Intelligence understand about [specific dimension] of this program's structural execution reality?"

The 9 questions:

| Object | Cognitive Question |
|--------|-------------------|
| structural_posture | What is this program's structural identity and qualification state? |
| tension_map | Where do independent structural risk vectors converge? |
| constraint_inventory | What structural truths limit this program's operational capacity? |
| exposure_assessment | Where is this program structurally vulnerable? |
| trajectory_assessment | What structural patterns are worsening, stable, or unmeasurable? |
| decision_surface | What structurally-derived leverage points exist, and what are their properties? |
| absence_profile | What did PI look for and not find? |
| competitive_intelligence | *See §3 — RECLASSIFIED* |
| operational_ceiling | What is this program's structural operational ceiling and what drives it? |

## 2. What Is NOT a Cognition Object?

### 2.1 Explicit Exclusions

| Category | Why Excluded | Example |
|----------|-------------|---------|
| **Pipeline artifact** | L0-L3 inputs to cognition, not cognition itself | Signal (L1), Condition (L2), Consequence (L3) |
| **Projection artifact** | Assumes audience, format, or rendering surface | Boardroom narrative, advisory memo, briefing slide |
| **Derived view** | Reorganization of existing cognition without new understanding | A filtered subset of constraint_inventory |
| **Supporting artifact** | Metadata, provenance, configuration — not structured cognition | CIP manifest, ProjectionConfig, rendering metadata |
| **Commercial framing** | Positions PI capabilities rather than describing specimen reality | Competitive positioning document, detection advantage list |
| **Temporal composite** | Comparison across pipeline runs — requires two PICPs, not one | Delta report, trend analysis, velocity decay measurement |
| **Consumer artifact** | Produced by projection consumers, not by PICR | Consumer-generated summary, reformatted excerpt |

### 2.2 Boundary Cases

| Case | Verdict | Reasoning |
|------|---------|-----------|
| A condition that happens to be structurally rich | NOT cognition | Conditions are L2 truth. Cognition is L4 synthesis FROM conditions |
| A consequence combination pattern | NOT cognition | Combinations are L3 composition rules. Cognition consumes them |
| A signal family summary | NOT cognition | Signal summaries are L1 measurement. They feed cognition objects |
| An urgency classification within decision_surface | IS cognition (part of decision_surface) | Urgency is rule-based from severity × effort_scope — a structural property, not a recommendation |
| A trajectory projection with temporal data | IS cognition (when deterministic) | Trajectory is a property of pattern type — STRUCT_GRAVITY_WELL inherently worsens because mass accumulation is self-reinforcing |
| A "what traditional tools miss" assessment | BORDERLINE — see §3 | Depends on framing: detection boundary = cognition, competitive advantage = projection |

## 4. Immutability, Versioning, and Governance

### 4.1 Immutability

**Cognition Objects are immutable within a single pipeline run.**

| Rule | Description |
|------|-------------|
| Production immutability | Once a materializer produces a cognition object, it is frozen. No in-place mutation |
| Package immutability | Once the PICP is assembled, no component may be altered, removed, or added |
| Projection immutability | PRE consumes cognition objects. It may select, compress, sequence, and render — but may not mutate the source object |
| Consumer immutability | Consumers receive cognition through PRE. They may not modify the underlying cognition object |
| Cross-run mutability | A new pipeline run produces a new PICP. The old PICP is not modified — both exist as independent artifacts. Comparison between them is valid (temporal cognition) |

**Mutation is prohibited at all layers:**

| Actor | May Produce | May Consume | May Mutate |
|-------|------------|-------------|------------|
| PICR materializer | YES (its own object) | CIP inputs | NO (other objects) |
| PICP assembler | YES (the package) | All 9 objects | NO (individual objects) |
| PRE | NO | PICP | NO |
| Consumer | NO | PRE output | NO |
| Marketplace module | NO (L4 cognition) | Module-specific inputs | NO |

### 4.2 Versioning

**The PICP is the versioned unit — not individual cognition objects.**

| Rule | Description |
|------|-------------|
| Version identity | Each PICP is identified by: specimen_id + run_id + pipeline_version |
| Object identity | Individual cognition objects are not independently versioned. They inherit the PICP version |
| Delta computation | Two PICPs from different runs of the same specimen can be structurally compared. The delta is a valid artifact (temporal cognition — future capability) |
| Schema versioning | If a cognition object's schema changes (fields added/removed), the PICP version increments. Backward-incompatible changes require a new PICP major version |

### 4.3 Governance

**Production governance:**
- Each cognition object is produced by exactly one materializer (pure function)
- Materializers have ZERO interpretive authority
- Materializers consume only CIP inputs — no external state, no audience context, no configuration beyond CIP
- Materializer correctness is verifiable by replay: same CIP → same object

**Assembly governance:**
- The PICP must contain ALL constitutional cognition objects (currently 9, pending reconstitution of competitive_intelligence → detection_boundary)
- No partial PICPs. A PICP missing any constitutional object is INVALID
- Order within the PICP is not semantically meaningful — objects are accessed by name, not position
- The PICP carries metadata: specimen_id, run_id, pipeline_version, production_timestamp, CIP_hash

**Consumption governance:**
- All 13 absolute prohibitions apply to every consumer of cognition objects
- Consumers may: render, hide, compress, expand, reframe, sequence, traverse, query, export
- Consumers may NOT: create cognition, modify cognition, exceed authority, fabricate evidence, soften qualification, rank without evidence
- These prohibitions are inherited from PI.PICP-STRATEGY-AND-CANONICALIZATION.01 and are NON-OVERRIDABLE

**Extension governance:**
- New cognition objects must pass the Qualification Test (see COGNITION_OBJECT_QUALIFICATION_TEST.md)
- Addition of a new cognition object is a G1 stream (architecture-mutating)
- Removal of a constitutional cognition object is a G1 stream
- Reclassification (A→D or D→A) is a G1 stream
- The qualification test itself may only be modified by a G1 stream

## 5. Constitutional Findings

### 5.1 The competitive_intelligence Reconstitution

This stream finds that `competitive_intelligence` as named and framed does not pass the Cognition Object qualification criteria. It fails audience-independence (commercial audience assumed) and projection-freedom (competitive positioning vocabulary).

The underlying data IS valid cognition. The reconstitution to `detection_boundary` preserves the cognitive content while removing the projection contamination.

**Action required:** When PICR is implemented, the materializer for this object should produce `detection_boundary`, not `competitive_intelligence`. The PICP schema should reflect this name. PRE may render the same information as "competitive intelligence" in projection surfaces — that is L5 vocabulary applied to L4 cognition.

**Impact on PICP baseline:** Minimal. The object's content does not change. The name and field labels change. The PICP remains at 9 objects. The cognitive question changes from "what does PI detect that others can't?" to "what aspects of this program's structural reality were previously unmeasurable?"

### 5.2 The decision_surface Boundary

decision_surface is the cognition object closest to the L4/L5 boundary. It passes all criteria today because:
- Interventions are structural properties of condition types (vocabulary lookup, not advice)
- Urgency is rule-based (severity × effort_scope), not prioritization
- All leverage points are presented without ranking or audience-specific emphasis

If any future modification introduces ranking, audience filtering, or prescriptive language ("you should"), the object has crossed into L5 and must be reconstituted.

### 5.3 The operational_ceiling Vocabulary Boundary

operational_ceiling.operational_experience.likely_symptoms contains condition-type vocabulary that describes operational effects. These are structural properties of condition types, not audience-specific narrative. They should be formalized as CONDITION_VOCABULARY.operational_symptoms to make the derivation chain explicit and prevent vocabulary drift toward narrative.
