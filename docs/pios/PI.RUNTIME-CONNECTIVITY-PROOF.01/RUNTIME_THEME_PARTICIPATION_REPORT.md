## 5. Root Cause Verdict

### **RC-A: Consequence Hierarchy Problem — PROVEN**

Runtime cognition cannot become a first-class theme contributor. The hierarchy blocks it:

1. Runtime conditions produce canonical atomic consequences (COORD_FRAG, DEP_AMP, etc.) at runtime loci (DOMAIN-03, DOMAIN-08, DOMAIN-11, etc.)
2. These atomics exist in `atomic_consequences` but NOT in `topLevel` consequences
3. They don't enter `topLevel` because:
   - At loci with ≥3 atomics, `detectCombinations` creates a SYSTEMIC_OP_FRAG or STAB_RISK combination, but the combination sources from all atomics at the locus — the combination's `source_conditions` reference the COMPOUND_CONVERGENCE condition (static), not the runtime conditions
   - At loci with <3 atomics, the runtime atomics are non-defining (`_defining: false`) and are excluded from topLevel by the filter at line 618: `if (!csq._defining) continue` (unless their locus is combined)
4. `consequence_themes` are built from `topLevel` consequences only
5. Therefore runtime conditions never appear in themes

**This is NOT a narrative consumption problem.** The hierarchy itself prevents runtime conditions from producing top-level consequences that could create or enrich themes.

## 6. The Exact Blocker

Line 617-619 in ConsequenceCompiler.js:

```javascript
for (const csq of deduped) {
    if (combinedLoci.has(csq._lk)) continue   // skip atomics at combined loci
    if (csq._defining) topLevel.push(...)      // only defining atomics
}
```

Runtime atomics are produced by `mapRuntimeCondition` which calls `makeAtomic(type, cond, scope, false, registry)` — the `false` means `_defining: false`. Non-defining atomics are excluded from topLevel unless their locus was claimed by a combination. And even when the locus IS combined, the runtime atomic is still excluded (it's at a combined locus, so the `continue` fires).

**The `_defining` flag and the `combinedLoci` exclusion jointly prevent runtime atomics from ever reaching topLevel.**

---

## 7. Recommended Next Action

Make runtime consequences eligible for topLevel participation:

**Option A:** Set `_defining: true` for runtime atomics in `mapRuntimeCondition`. This allows runtime atomics at uncombined loci to appear in topLevel directly as standalone consequence findings.

**Option B:** Build consequence themes from `allAtomics` (or `deduped`) instead of `topLevel` only. This would include all consequence evidence in the theme classification, including runtime atomics that were excluded from topLevel.

**Option C:** Make `detectCombinations` produce combination consequences that explicitly track runtime contributor conditions, not just COMPOUND_CONVERGENCE.

Option A is the simplest. Option B is the most architecturally correct (themes should reflect the full consequence picture, not just the topLevel filter). Option C is the deepest but highest risk.

Recommended: **Option A first** — verify runtime consequences appear in topLevel and themes. Then evaluate if Option B is needed.
