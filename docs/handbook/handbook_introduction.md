# PROGRAM INTELLIGENCE HANDBOOK
Handbook Introduction

## Purpose

This chapter introduces the Program Intelligence discipline and explains the purpose of the handbook.

The handbook serves as the structured publication layer of the Program Intelligence project. It explains the discipline as a coherent system of knowledge while referencing the authoritative artifacts produced across the project streams.

The handbook does not redefine those artifacts. Instead, it organizes them into a readable structure that explains how the discipline works as a whole.


## WHAT PROGRAM INTELLIGENCE IS

Program Intelligence is a discipline focused on understanding how programs actually execute.

Programs generate large volumes of operational data through delivery systems such as:

- issue tracking systems
- engineering repositories
- deployment platforms
- operational platforms
- delivery governance tools

Traditionally, this data remains fragmented and operational in nature. It describes activity, but it rarely provides insight into the condition of the program itself.

Program Intelligence addresses this gap.

The discipline interprets observable execution evidence and transforms it into execution signals that reveal the condition of program delivery.

These signals allow organizations to understand:

- delivery stability
- risk formation
- execution pressure
- program acceleration or slowdown
- structural delivery weaknesses

The result is a new capability: observing programs as dynamic execution systems.


## THE EVIDENCE-FIRST PRINCIPLE

The Program Intelligence discipline operates under a strict principle:

**Evidence First.**

All insight must originate from observable execution evidence.
Claims about program health, risk, or performance must always be traceable to signals derived from real execution data.
This principle ensures that Program Intelligence remains grounded in observable program behavior rather than subjective reporting.


## THE ROLE OF SIGNALS

- Signals are the core mechanism of the discipline.
- Execution signals are structured interpretations of execution evidence.

They allow patterns within program activity to become observable.
Signals transform raw operational data into interpretable indicators of program conditions.

Through signals, the discipline can observe:

- stability
- volatility
- delivery pressure
- execution imbalance
- structural risk formation


## THE PROGRAM INTELLIGENCE STACK

The Program Intelligence discipline is organized into multiple conceptual layers.
Each layer explains a different aspect of the discipline.
The Krayu project organizes these layers through dedicated streams.

These streams produce the authoritative artifacts that define the discipline.


## DISCIPLINE STRUCTURE IN THE KRAYU PROJECT

The Program Intelligence discipline is developed through six streams.


- 10 — Program Intelligence Discipline : **defines the doctrinal foundations** of the discipline.
It establishes the purpose, language, and principles of Program Intelligence.
- 20 — Program Intelligence Framework : **defines the conceptual framework** of the discipline.
It explains how execution evidence becomes interpretable intelligence.
- 40 — Signäl Execution Signal Infrastructure : **defines the signal science** of the discipline.
It explains how execution signals are constructed and interpreted.
- 50 — Program Intelligence Demonstration : **defines controlled demonstration environments** used to illustrate the discipline in action.
- 60 — Program Intelligence Case Studies : **analyzes real program** environments using the discipline.
- 30 — Program Intelligence Commercialization : **explains** how the discipline becomes a commercial advisory and product offering.


## THE ROLE OF THE HANDBOOK

The handbook does not replace the work of these streams.
Instead, it serves as the navigation layer of the discipline.

The handbook:

• explains the discipline as a whole
• connects the outputs of the streams
• guides the reader through the intellectual structure of Program Intelligence
• presents the discipline as a coherent publication

The handbook can be read in two ways.

## DISCIPLINE INTENT

Program Intelligence aims to create a new capability for organizations: observing programs as living execution systems.

By transforming execution evidence into signals and signals into intelligence, the discipline allows leadership to understand program conditions with far greater clarity. This capability represents a new form of operational observability for program execution.
