PROGRAM INTELLIGENCE HANDBOOK STRUCTURE

Purpose

This artifact defines the official structure of the Program Intelligence Handbook.

The handbook is the synthesis and navigation layer of the Program Intelligence discipline.
It consolidates the intellectual outputs produced across the Program Intelligence project into a coherent publication.

The handbook does not redefine authoritative stream artifacts.
It references, connects, and explains them as an integrated body of knowledge.


HANDBOOK ROLE

The Program Intelligence Handbook serves five functions:

• provide the reader with a structured path through the discipline
• synthesize outputs from the authoritative streams
• explain how the parts of the discipline fit together
• establish the publication structure of Program Intelligence
• act as the official navigation layer of the discipline


HANDBOOK GOVERNANCE

The handbook is governed by Stream 05 — Program Intelligence Handbook.

Stream 05 may produce only handbook-specific artifacts such as:

• handbook structure documents
• chapter drafts
• synthesis artifacts
• overview narratives
• navigation artifacts
• conceptual maps supporting handbook assembly

Stream 05 must not redefine the authoritative content owned by the other streams.

Authority remains distributed as follows:

• Stream 10 — Program Intelligence Discipline
• Stream 20 — Program Intelligence Framework
• Stream 40 — Signäl Execution Signal Infrastructure
• Stream 50 — Program Intelligence Demonstrations
• Stream 60 — Program Intelligence Case Studies
• Stream 30 — Program Intelligence Commercialization
• Stream 00 — Governance


HANDBOOK SOURCE MODEL

The handbook is assembled from authoritative artifacts produced in the other streams.

Each handbook part references and synthesizes source artifacts from its corresponding stream.

The handbook therefore acts as a structured publication layer above the stream artifacts.

HANDBOOK NAVIGATION RULE

Each handbook chapter should clearly indicate:

• its purpose in the handbook
• which stream it synthesizes
• which authoritative artifacts it references
• how it connects to the preceding and following handbook sections

This ensures the handbook remains a navigation and synthesis layer rather than a replacement for source artifacts.


ARTIFACT RELATIONSHIP RULE

Authoritative source artifacts remain in their original stream directories.

The handbook must reference them from the handbook layer.

No source artifact should be moved into the handbook directory.
No handbook artifact should be stored in another stream directory.

This preserves separation between:

• source authority
and
• synthesis authority
