# Execution Signal Taxonomy

Execution signals are grouped into canonical families.

## Delivery Stability
Signals that detect deterioration in delivery predictability.

Examples:
- schedule volatility
- milestone slippage
- release drift

## Dependency Health
Signals that detect cross‑team coordination stress.

Examples:
- unresolved dependency backlog
- integration failures

## Scope Integrity
Signals detecting instability in program scope.

Examples:
- requirement churn
- backlog volatility

## Quality Integrity
Signals revealing deterioration in engineering health.

Examples:
- defect escalation
- CI pipeline instability

## Execution Throughput
Signals observing delivery momentum and capacity.

Examples:
- throughput variance
- cycle time increase

## Governance Health
Signals identifying governance weaknesses.

Examples:
- decision latency
- escalation delay

## Program Confidence
Signals representing confidence in expected outcomes.
