# Framework–Signal Alignment

## Purpose

This artifact defines the canonical alignment between:

• Program Intelligence Framework constructs (Stream 20)
• Signäl Execution Signals (Stream 40)

The purpose of this artifact is to ensure consistent interpretation across the
Program Intelligence analytical framework and the execution signal layer.

Framework artifacts remain unchanged. This document provides the mapping layer
that connects analytical constructs with their underlying execution signals.

This artifact ensures:

• signal-driven interpretation
• Evidence‑First compliance
• structural consistency between framework and signal infrastructure

---

# Alignment Model

The alignment follows the structure:

Framework Construct  
→ Signal Family  
→ Signals  
→ Interpretation Relationship

Framework constructs interpret signal families.  
Signal families are composed of individual execution signals.

The framework therefore **interprets signals but does not redefine them**.

Signal definitions remain authoritative in:

Stream 40 — Signäl Execution Signal Infrastructure

# Governance Principle

The following separation of responsibility is maintained:

Stream 40 — Execution Signals  
Defines observable program behaviour.

Stream 20 — Program Intelligence Framework  
Interprets those signals to produce execution intelligence.

The alignment artifact ensures both layers remain structurally consistent
without modifying the canonical framework artifacts.
